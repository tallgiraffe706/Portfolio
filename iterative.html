Iterative Design Project

For this project, our team created a user interface for a Y-Combinator startup named OnScale. OnScale seeks to connect creators with brands so that the creators can spend more time creating. With this premise in mind, our team created concepts for webpages, then wireframes for a possible app, and finally, a high fidelity mock up to test with real users.

Sketches & Wireframes

OnScale is a Y-Combinator startup seeking to connect small creators to brands. We chose to make our design of OnScale on a desktop application. This decision is because creators, who this app is target towards, do most of their content creation such as editing, streaming, etc, on desktop computers.

To kick off our redesign of the app, we each completed our own LoFi sketch based on our personal understanding of OnScale to generate as many ideas as possible. In total, we ended up with four sketches to work off of:

The Mockups

Interactive Hi-Fi Mockup

First, we created an interactive hifi mockup of our app design using the combined lofi design. Then, we presented our mockup to our class and an industry professional for critique.

Feedback & Fixes

We identified four main pieces of feedback from the critique. For each of these pieces, we discussed it together as a group before implementing a comprehensive fix to address it:

  1. Feedback: Make it clear how to go back to the previous page.
    • Fix: Added a breadcrumb to the top of each subpage, allowing users to easily navigate back to the main page they came from.
  2. Feedback: Horizontal scrolling elements are difficult to interact with using a mouse.
    • Fix: Replaced them with buttons, allowing users to simply click the buttons to make the element scroll rather than having to scroll themselves.
  3. Feedback: Add a partnership timeline to distinguish past from present partnerships.
    • Fix: Introduced a new timeline view to the partnerships section that shows when partnerships took place.
  4. Feedback: Provide more information about brands, including why they are recommended to the user.
    • Fix: Added a TLDR for each partnered brand on the profile page and a brief explanation beneath each recommended brand.

User Testing Main Takeaways

We had the following feedback from our three users:

User 1:
  1. Unclear horizontal scrolling mechanism.
  2. Unclear WHICH referral codes the graph is talking about.
  3. Distinction between currently partnered vs former partners vs recommended partners is unclear.
  4. Seeing actual partnerships is a bit complicated due to the lack of labels and logos on the partners (have a way to get to negotiations from partnerships page).
  5. Where to find initial offer unclear, reviewer expected it to be in chronological order but it is reverse chronological, formatting is too generic/unclear. Connection drawn between similarly styled elements that are also used in brand negotiation listing which themselves don't have a chronological connection.
User 2:
  1. First thing that stands out is the SquidGamer logo and the colors of the UI, but would be nice to have a button to edit profile ./profile settings (Settings could be added to navigation menu).
  2. Doesn’t know how past/current brands are differentiated, so we should find more ways to differentiate current/former partnerships (fading/size?/ etc…).
  3. Profile page is nice thing to use analogous to a homepage or a dashboard that shows all of the information.
  4. Horizontal scrolling rectangles might be a bit large - (maybe make two rows of smaller rectangles and also be sure to add arrows for horizontal scrolling).
  5. Confusion with where to find initial offer and why to contact OnScale on negotiations page, not very obvious and intuitive.
User 3:
  1. Distinguish what is viewable just to you vs. brands, unclear (Distinguish sections).
  2. Slide scroll is nice, always ensure that its cut off at the edge which imply side scroll.
  3. Left navigational static banner is amazing.
  4. The information displayed in the negotiation history is good for each time segment, except like said before, make it very clear that it is flowing nicely.
  5. Payments: 10-99s specific/documents maybe list those out.

Changes Based on User Testing

Using the feedback from user testing, we would want to make the following changes in the future:

  1. Graph should more explicitly show the type of referral codes being displayed (referral codes for *all* companies, referral codes for X types of products, etc…) and present buttons/filter options explicitly.
  2. Have a way of distinguishing between former/current partners(different sections, labels, OR former partners are “grayed” out) and ongoing vs. completed negotiations.
  3. Add a button to navigate directly from partners page to the negotiations page (or perhaps a “see negotiation with X partner” button for each partner).
  4. Turn the negotiation history page into a timeline-based format (current styling/format mimics the non-chronological negotiation overviews too closely).
  5. Add a settings page to edit bio and profile picture. Put the navigation link to the settings page in the sidebar under payments.
  6. Add an explanation/description to explain purpose of the Contact Onscale button on the negotiations page, and/or separate into many buttons which end up contacting OnScale anyway (“Click here for help, something doesn’t look right, etc…”).
  7. Under payments section, have an option to see specifically and list out how taxes are being calculated.
